要让网页能够实时更新数据，可以结合 Flask 和 JavaScript 实现。以下是一个完整的示例代码，它展示了如何通过 Flask 提供一个实时更新的数据接口，并使用 JavaScript 在前端自动更新数据。

python
from flask import Flask, jsonify
import random

app = Flask(__name__)

# 模拟一个动态生成的地址
def get_random_address():
    cities = ["北京", "上海", "广州", "深圳", "杭州"]
    districts = ["朝阳区", "黄浦区", "天河区", "南山区", "西湖区"]
    return f"{random.choice(cities)}市{random.choice(districts)}"

@app.route('/get_address')
def get_address():
    # 返回一个随机生成的地址
    address = get_random_address()
    return jsonify({"address": address})

@app.route('/')
def home():
    return '''
        <!DOCTYPE html>
        <html lang="zh">
        <head>
            <meta charset="UTF-8">
            <meta name="viewport" content="width=device-width, initial-scale=1.0">
            <title>实时更新地址示例</title>
        </head>
        <body>
            <h1>公司地址</h1>
            <p id="address">正在加载地址...</p>

            <script>
                function updateAddress() {
                    fetch('/get_address')
                        .then(response => response.json())
                        .then(data => {
                            document.getElementById('address').textContent = "地址: " + data.address;
                        });
                }

                // 每5秒更新一次地址
                setInterval(updateAddress, 5000);

                // 页面加载时更新一次地址
                updateAddress();
            </script>
        </body>
        </html>
    '''

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000, debug=True)

2. 代码解释

  • get_random_address 函数：生成一个随机的城市和区组合作为地址。
  • /get_address 路由：当前端请求时返回一个 JSON 格式的随机地址数据。
  • 主页路由 /：返回一个包含 HTML 和 JavaScript 的页面，JavaScript 会每隔5秒从 /get_address 接口获取新的地址并更新网页上的内容。
  • JavaScript 代码：
    • fetch('/get_address')：请求 Flask 应用的 /get_address 路由获取最新的地址。
    • setInterval(updateAddress, 5000)：设置一个间隔，定期更新地址（每5秒）。

from flask import Flask, jsonify
import random

app = Flask(__name__)

# 模拟一个动态生成的地址
def get_random_address():
    cities = ["北京", "上海", "广州", "深圳", "杭州"]
    districts = ["朝阳区", "黄浦区", "天河区", "南山区", "西湖区"]
    return f"{random.choice(cities)}市{random.choice(districts)}"

@app.route('/get_address')
def get_address():
    address = get_random_address()
    return jsonify({"address": address})

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000, debug=True)

解决方法

这里有几种解决这个问题的方法：

1. 在服务器端启用 CORS 支持

如果你可以控制 API 服务器（例如你使用的 Flask 服务器），你需要在服务器端启用 CORS 支持，以允许来自 https://wangwenyan010928.github.io 的请求。

在 Flask 中，你可以使用 flask-cors 扩展来实现这一点。以下是如何修改 Flask 代码：

python
from flask import Flask, jsonify
from flask_cors import CORS  # 引入 CORS

app = Flask(__name__)
CORS(app)  # 启用 CORS 支持

def get_random_address():
    cities = ["北京", "上海", "广州", "深圳", "杭州"]
    districts = ["朝阳区", "黄浦区", "天河区", "南山区", "西湖区"]
    return f"{random.choice(cities)}市{random.choice(districts)}"

@app.route('/get_address')
def get_address():
    address = get_random_address()
    return jsonify({"address": address})

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000, debug=True)

通过添加 CORS(app)，你将允许来自所有来源的请求。如果你只想允许特定来源的请求，例如你的 GitHub Pages 站点，可以使用 CORS(app, origins=["https://wangwenyan010928.github.io"])。

2. 使用 no-cors 模式

你可以在 fetch 请求中使用 no-cors 模式，但这有很大的限制，可能无法满足你的需求。具体做法如下：

3. 使用代理服务器

如果你无法控制 API 服务器，也无法直接修改 CORS 设置，你可以考虑使用一个代理服务器来解决这个问题。代理服务器可以位于同一个域下，从而避免跨域问题。

4. 在客户端加载数据时处理 CORS

如果你没有办法修改服务器端的代码，最终你可能需要寻求使用公共 API 或托管你自己的 API，在允许跨域请求的服务器上。
